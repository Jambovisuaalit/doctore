# SKILL: Risk Management

> Purpose: Convert positive EV into a survivable stake while controlling model uncertainty, correlation, drawdown, and portfolio concentration.

## Core principle

A positive-EV estimate is not enough. Stake sizing must assume the probability estimate is imperfect.

## Correct Kelly formula

For decimal odds `d` and probability used for sizing `p_sized`:

```text
full_kelly_fraction = (p_sized * d - 1) / (d - 1)
```

Equivalent form with net odds `b = d - 1` and loss probability `q = 1 - p_sized`:

```text
full_kelly_fraction = (b * p_sized - q) / b
```

If the result is zero or negative, stake is zero.

## Probability used for sizing

Do not automatically size from the headline model probability.

Recommended process:

```text
model_edge = model_p - market_no_vig_p
sizing_p = market_no_vig_p + shrinkage_factor * model_edge
```

Starting shrinkage factors:

| Model status | Factor |
|---|---:|
| New or uncalibrated | 0.25–0.50 |
| Partially validated | 0.50–0.75 |
| Validated and stable | 0.75–1.00 |

Use a lower factor when data quality is degraded, the market is illiquid, the price is stale-prone, or the prediction is outside the training domain.

## Fractional Kelly tiers

| Status | Maximum fraction of full Kelly |
|---|---:|
| New or uncalibrated model | 0.10 |
| Partially validated model | 0.25 |
| Validated model with stable CLV | 0.25–0.40 |
| Maximum permitted exceptional tier | 0.60 |

The 0.60 tier is permitted only when all apply:

- at least 500 comparable settled bets for the current model family;
- stable out-of-sample calibration;
- positive CLV across recent and full validation windows;
- no material model drift;
- liquid market and reliable price execution;
- current drawdown below warning level;
- no correlated concentration breach.

## Stake calculation

```text
raw_fraction = max(0, full_kelly_fraction) * allowed_kelly_fraction
raw_stake = bankroll * raw_fraction
final_stake = minimum of:
  raw_stake
  bankroll * max_bankroll_per_bet
  remaining_open_exposure_capacity
  remaining_daily_turnover_capacity
  remaining_league_capacity
  remaining_rolling_3d_capacity
  sportsbook_limit
```

Round down to the permitted stake increment. Never round up above a risk cap.

## Default Doctore limits

```yaml
max_bankroll_per_bet: 0.02
max_open_exposure: 0.10
max_daily_turnover: 0.25
max_league_exposure: 0.15
max_rolling_3d_turnover: 0.40
drawdown_warning: 0.10
drawdown_review: 0.15
drawdown_pause: 0.20
```

Definitions:

- `open exposure`: unsettled risk currently at stake;
- `daily turnover`: total stakes placed during the day, including settled positions;
- `league exposure`: open and recent concentration defined by portfolio policy;
- `rolling 3d turnover`: total stakes over the trailing three days.

## Unit conversion

When `1 unit = 500 EUR`:

```text
units = final_stake / 500
```

Units are a reporting convenience. Percentage-of-bankroll caps remain authoritative.

## Correlation controls

Group related candidates before sizing.

Minimum correlation groups:

- same event;
- same team or player;
- same starting pitcher, goalie, quarterback, or key injury assumption;
- moneyline, spread, total, team total, and props with shared drivers;
- multiple matches exposed to the same weather or scheduling shock;
- multiple prices generated by the same model failure mode.

Default handling when correlation is not quantified:

- treat strongly related positions as one combined position;
- cap the group at the single-bet limit or stricter event limit;
- reduce each position proportionally;
- block parlays and same-game parlays unless joint probability is modeled.

## Drawdown controls

Measure drawdown from the bankroll high-water mark.

| Drawdown | Action |
|---|---|
| <10% | Normal configured sizing. |
| 10–15% | Reduce allowed Kelly fraction by 25%; audit recent CLV. |
| 15–20% | Reduce allowed Kelly fraction by 50%; pause weak markets and new models. |
| >=20% | Pause new betting and perform formal model, data, and execution review. |

A pause is not a claim that variance cannot explain the drawdown. It is a risk-control response to uncertainty.

## Stop conditions

Set stake to zero when:

- model or data status is blocked;
- Kelly is non-positive;
- target EV disappears at the executable price;
- portfolio state is stale;
- a risk cap is already reached;
- correlation cannot be assessed;
- a model is in a declared rollback or drift state;
- behavior shows loss chasing or unauthorized size escalation.

## Parlays and accumulators

Do not multiply marginal probabilities unless independence is justified.

Required for any parlay:

- modeled joint probability or defensible dependence structure;
- complete combined odds;
- combined EV calculation;
- correlation adjustment;
- separate strict stake cap.

Without these, return `BLOCKED`.
